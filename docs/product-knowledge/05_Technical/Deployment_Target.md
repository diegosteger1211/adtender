# Deployment Target вЂ” adtender

---

## 1. Target Domain

| Environment | URL | Purpose |
|---|---|---|
| Production | `https://adtender.adesso-consulting.de` | Live system |
| Staging | `https://staging.adtender.adesso-consulting.de` | Pre-production validation |
| Preview | `https://<branch>.adtender.pages.dev` | Per-PR preview deployments (auto-generated by Cloudflare Pages) |
| Local | `http://localhost:5173` (frontend) В· `http://localhost:8787` (API) | Developer workstation |

> **DNS requirement:** `adtender.adesso-consulting.de` must be a CNAME pointing to the Cloudflare Pages deployment. The `adesso-consulting.de` zone must be managed in Cloudflare DNS.

---

## 2. Hosting Model

adtender uses a **split-deployment model** on Cloudflare:

| Layer | Technology | Deployment unit |
|---|---|---|
| Frontend (SPA) | React + Vite в†’ static assets | Cloudflare Pages |
| Backend (API) | Cloudflare Workers (TypeScript) | Cloudflare Workers |
| Database | Cloudflare D1 (SQLite-compatible) | Bound to Workers |
| Key-Value Store | Cloudflare KV | Session tokens, rate limiting |
| Email | Resend | Called from Workers |

> adtender uses Cloudflare D1 as its database вЂ” bound to the Worker, no external connection string required.

## 5. Custom Domain Setup

1. `adesso-consulting.de` zone must be in Cloudflare (not just pointed at Cloudflare)
2. In Cloudflare Pages settings в†’ Custom domains в†’ add `adtender.adesso-consulting.de`
3. Cloudflare auto-provisions the TLS certificate (Let's Encrypt via Cloudflare)
4. DNS record: `adtender CNAME <project>.pages.dev` (Cloudflare manages this automatically when added via Pages UI)

---

## 6. Multi-Environment Strategy

| Branch | Deploys to | Environment variables |
|---|---|---|
| `main` | Production | `.env.production` / Cloudflare Pages secrets |
| `staging` | Staging | Separate Pages project or Pages branch deploy |
| `feature/*` / PR | Preview | Cloudflare Pages preview URL (auto) |

Preview deployments use the same production Worker (or a dedicated staging Worker вЂ” see `Cloudflare_Deployment.md`).
